The best way to understand how COUIJS works is to take a look at the two exmples provided (config[1,2].html and config[1,2].xml)...


The XML config file needs to be organized using a three levels hierarchie:
  • section
  • subsection
  • key
Sections and Sub-sections are mandatory in the XML configuration file  (see an example here):
  • there should be at least one section;
  • there should be at least one sub-section in each section;
  • also, there should be at least one key in each sub-section.
Mandatory tags:
  • name
  • label
  • type
  • value
The name should be unique.
The name is used in rules (either enable or visible rule") to reference this particular key.

Example:

...
<KEY name="IsAudioEnabled"
    label="Enable Audio Recording"
    help="Enable Audio Recording"
    type="boolean"
    value="true">
</KEY>
<KEY name="AudioSampleRate"
    label="Sample rate"
    suffix="Hz"
    help="Sample rate, in Hz"
    type="list"
    value="16000"
    index="2"
    values="['8000','11025','16000','22050']"
    visible="($IsAudioEnabled)">
</KEY>
...

The label is a short text, which is displayed on the keft of the iput field
The name is used in rules (either enable or visible rule") to reference this particular key.

Example:

<KEY name="VideoH264Profile"
    label="H.264 Profile"
    help="H.264 Profile : BP=Baseline - MP=Main - HP=High"
    type="list"
    value="High"
    index="2"
    values="['Baseline', 'Main', 'High']">
</KEY>

Optional tags:
  • help
  • suffix
  • advanced
  • hidden
  • visible
  • enable
  • skipline
Help: This is an optional tag, and it can be used for both a Section, Subsection, or a Key. The text described in the help tag is displayed as a tooltip when the mouse cursor moves over the displayed field. Example: Advanced: This is an optional tag, and it can be used for both a Section, Subsection, or a Key.
