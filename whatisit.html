COUIJS is a minimalist tool which
  • is written in Javascript,
  • read an XML configuration file,
  • and create dynamically a user-interface which allow to edit and update the configuration file.
The typical use of COUIJS would be in an embedded device, where the configuration is stored as an XML file.
You could change the configuration by using one of these methods:
  • log into the device (for instance with ssh), then use vi or something similar (nano, etc.)
  • create a custom UI in javascript
  • or use COUIJS ;-)
The 1st option (vi or nano) is certainly not an option for the end-user.

The 2nd option has the advantage of offering a great level of customization.
Unfortunately, every time you change the XML configuration file (adding, removing or changing something), you will need to change this custom program.

The 3rd option, using COUIJS, offer much less customization, but since it builds the User Interface solely from the XML configuration file, it does not need to change.

The concept behind COUIJS is to add a few information within the XML configuration file so the tool can create the UI for you!
These information will be typically:
  • the type of input (checkbox, list, etc.)
  • some information related to the type of input (for instance, if it is a list, that would be the list of possible of values)
  • and some optional rules (for instance you might want to edit some settings only if another setting has a specific value).
COUIJS needs the XML configuration file to be organized following a specific hierarchical structure, grouping the different settings into
  • sections
  • sub-sections
  • keys
The keys being the only editable items by the user of the UI, and the sections and sub=-sections being a way to organize the items that can be configured.

Also, COUIJS is using the DOJO toolkit.
This way, it can uses the nice widgets provided by Dojo (dijit).
